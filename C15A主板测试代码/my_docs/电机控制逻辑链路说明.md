# 电机控制逻辑链路说明

## 1. 文档目的

本文档用于梳理当前工程中电机控制相关的完整逻辑链路，帮助后续做代码重构。

重点说明内容包括：

- 程序从启动开始的执行路径
- 电机控制主链路
- 定时器中断如何驱动闭环
- 串口目标值如何进入控制系统
- 编码器反馈如何回到 PID
- PWM / 方向控制最终如何落到硬件引脚
- 当前涉及到的函数、API、变量、文件路径

当前分析基于现有 C / D 双电机控制版本。

---

## 2. 当前控制对象

当前主控制对象已经收敛为两个电机通道：

- 电机 C
- 电机 D

其中：

- C 电机编码器走 `TIM3`
- D 电机编码器走 `TIM4`
- C 电机 PWM 走 `TIM8->CCR1`
- D 电机 PWM 走 `TIM8->CCR3`

---

## 3. 整体控制链总览

整个电机控制主链可以概括为：

```text
程序启动
  -> 硬件初始化
  -> 编码器 / PWM / 电机方向使能初始化
  -> 定时控制中断启动
  -> 主循环只负责显示与调试输出

串口输入目标值
  -> 写入 Target_EncoderC / Target_EncoderD

定时中断触发
  -> 获取姿态
  -> 扫描按键
  -> 读取编码器反馈值
  -> 判断是否允许电机输出
  -> 执行 PID
  -> PWM 限幅
  -> 设置方向脚 + PWM 占空比
  -> 电机转动
```

## 6. 定时器中断驱动的控制核心

当前电机控制主心跳入口位于：

- `MiniBalance/CONTROL/control.c:11`

函数：

- `TIM1_UP_IRQHandler(void)`

这是整个电机闭环控制的核心执行函数。

### 6.1 中断执行主流程

每次进入该中断时，当前执行流程如下：

1. 检查并清除中断标志
2. 翻转 `Flag_Target`
3. 处理延时标记 `delay_flag`
4. 获取姿态信息
   - `Get_Angle()`
   - 位置：`MiniBalance/CONTROL/control.c:21`
5. 扫描按键
   - `Key()`
   - 位置：`MiniBalance/CONTROL/control.c:22`
6. 在 `Flag_Target == 1` 时读取反馈值
7. 判断是否允许电机输出
   - `Turn_Off(Voltage)`
   - 位置：`MiniBalance/CONTROL/control.c:32`
8. 若允许输出，则执行 PID
9. 对 PWM 做限幅
10. 将结果下发到电机驱动

当前已支持：

- `TC:<rpm>`

  - 位置：`SYSTEM/usart/usart.c:159`
- `td_ptr = strstr(Debug_Cmd_Buffer,"TD:")`

## 9. PID 控制链路

在定时中断中，当 `Turn_Off(Voltage) == 0` 时进入闭环控制：

- 位置：`MiniBalance/CONTROL/control.c:34`

函数定义：

- `Incremental_PIC(int Encoder, int Target)`
- 位置：`MiniBalance/CONTROL/control.c:122`

### 9.2 D 电机 PID

- `Motor_PwmD = Incremental_PID(Current_EncoderD, Target_EncoderD)`
- 位置：`MiniBalance/CONTROL/control.c:35`

函数定义：

- `Incremental_PID(int Encoder, int Target)`
- 位置：`MiniBalance/CONTROL/control.c:133`

### 9.3 当前 PID 输入输出含义

输入：

- `Target_EncoderX`：目标编码器量
- `Current_EncoderX`：当前编码器反馈量

输出：

- `Motor_PwmX`：当前电机 PWM 控制输出

说明：

- 当前控制结构本质上是“编码器反馈闭环速度控制”

- 进入 PID 前已经换算成 encoder 目标域

## 18. 当前架构存在的耦合点

如果你后续准备重构，当前代码最明显的耦合点有以下几个：

### 18.1 中断里职责过多

`TIM1_UP_IRQHandler()` 当前同时承担：

- 姿态更新
- 按键处理
- 编码器采样
- 电压采样统计
- 停机判断
- PID 计算
- 输出下发

建议后续拆成多个小函数。

### 18.2 目标输入与控制变量直接耦合

串口层现在直接写：

- `Target_EncoderC`
- `Target_EncoderD`

建议后续引入更清晰的目标结构体或命令层缓存。

### 18.3 用户输入单位与控制单位混杂

虽然现在已经把 rpm 换算放在输入边界，但工程里仍同时存在：

- rpm 语义
- encoder 目标语义
- 旧二进制帧目标语义

建议后续明确只保留一种用户接口语义。

### 18.4 电机通道仍未完全抽象

当前 C / D 两路仍靠散落变量组织：

- `Target_EncoderC`
- `Target_EncoderD`
- `Current_EncoderC`
- `Current_EncoderD`
- `Motor_PwmC`
- `Motor_PwmD`

建议后续抽象为通道结构体。

## 19. 建议的重构方向

如果后续你要重构当前工程，我建议优先按下面顺序推进。

## 19.1 第一阶段：梳理流程函数

把 `TIM1_UP_IRQHandler()` 中的逻辑拆成几个函数，例如：

- `ControlLoop_UpdateSensors()`
- `ControlLoop_UpdateFeedback()`
- `ControlLoop_CheckSafety()`
- `ControlLoop_RunPid()`
- `ControlLoop_ApplyOutput()`

这样先把流程读顺。

## 19.2 第二阶段：抽象电机通道结构体

例如：

```c
typedef struct
{
    int target_encoder;
    int current_encoder;
    int pwm;
    int (*read_encoder)(void);
    void (*apply_output)(int pwm);
} MotorChannel;
```

这样 C / D 的逻辑就能统一。

## 19.3 第三阶段：拆开输入层和控制层

建议把：

- 串口输入解析
- rpm 映射
- 控制目标更新

独立成一个单独模块，避免继续堆在 `usart.c` 中。

## 19.4 第四阶段：统一调试接口

保留一个统一调试输出层，固定输出：

- 目标值
- 反馈值
- PWM
- 使能状态

这样最利于回归验证。

---

## 20. 最终总结

当前电机控制链路的本质是：

1. `main()` 完成电机 / 编码器 / 串口 / 定时器初始化
2. 串口输入目标 rpm 或旧协议目标值
3. 写入 `Target_EncoderC / Target_EncoderD`
4. 定时中断 `TIM1_UP_IRQHandler()` 周期运行
5. 中断里读取 `Current_EncoderC / Current_EncoderD`
6. 进入 `Incremental_PIC / Incremental_PID`
7. 输出 `Motor_PwmC / Motor_PwmD`
8. `Set_Pwm()` 将其映射为方向脚 + PWM 比较寄存器输出
9. 电机执行动作

换句话说，当前最关键的控制主链是：

```text
目标输入 -> Target_EncoderX -> 定时中断 -> Read_Encoder -> PID -> Motor_PwmX -> Set_Pwm -> 电机输出
```

如果后续你准备重构，这条链建议作为最核心的“第一性主线”，其余显示、姿态、按键、调试都围绕这条主线逐步解耦。
